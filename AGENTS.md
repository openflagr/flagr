# Build & Test Commands
- `make build` - Build flagr server binary
- `make build_ui` - Build the Vue.js UI
- `make test` - Run all tests with coverage
- `make verify_lint` - Run golangci-lint
- `make verify_swagger` - Validate swagger spec
- `make gen` - Regenerate swagger files (api_docs + swagger)
- `make deps` - Install build dependencies (swagger, golangci-lint)
- `go test -run TestName ./pkg/...` - Run specific test
- `go test -bench . ./pkg/...` - Run benchmarks

# Code Style
- Go 1.25+ (use go-swagger for code gen, golangci-lint for linting)
- Group imports: std, internal, external (blank lines between)
- Interface-first design with factory funcs (e.g., NewEval())
- Use gorm.Model in entity structs, GORM for DB operations
- Errors returned explicitly, not wrapped unless needed
- JSON/GORM tags on struct fields
- Log with logrus (logrus.Infof)
- API responses use middleware.Responder
- DO NOT edit swagger_gen/ files (auto-generated by go-swagger)
- Test files: *_test.go, use testify assertions

# Project Structure
- `pkg/config/` - Configuration management and middleware
- `pkg/entity/` - Database models (Flag, Segment, Constraint, etc.)
- `pkg/handler/` - API request handlers (CRUD operations, evaluation)
- `pkg/mapper/` - Data mapping between entity and API models
- `pkg/mapper/entity_restapi/` - Entity to REST API model converters
- `pkg/notification/` - Notification handlers (email, slack, webhooks)
- `pkg/util/` - Utility functions
- `swagger/` - OpenAPI/Swagger specification files (YAML)
- `swagger_gen/` - Auto-generated Go code from swagger (DO NOT EDIT)
- `browser/flagr-ui/` - Vue.js frontend

# Database Support
- SQLite (github.com/glebarez/sqlite)
- MySQL (gorm.io/driver/mysql)
- PostgreSQL (gorm.io/driver/postgres)
- Database configured via env vars or CLI flags
- Use GORM for all DB operations

# API Development
- API spec defined in `swagger/*.yaml`
- After modifying swagger specs, run `make gen` to regenerate code
- The `swagger_gen/restapi/configure_flagr.go` is preserved during regeneration
- API operations implemented in `pkg/handler/`
- Models generated in `swagger_gen/models/`

# Testing
- Unit tests co-located with source code in `*_test.go` files
- Use testify for assertions (assert.Equal, assert.NoError, etc.)
- Test fixtures in `pkg/entity/fixture.go`
- Integration tests in `integration_tests/` using Docker Compose

# Evaluation Engine
- Flag evaluation logic in `pkg/entity/evaluator.go`
- Supports: boolean flags, variant flags, A/B testing
- Evaluation contexts include entityID, entityType, entityContext
- The evaluation is all done in memory with `eval_cache`
